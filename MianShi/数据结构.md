### B树的概念

```
	m: 表示一个树节点最多有多少个查找路径

1. 每个节点最多有m-1个关键字
2. 根节点最少可以有1个关键字
3. 非根节点最少有m/2个关键字
4. 每个节点的关键字都是按照从小到大的顺序排列, 每个关键字的左子树的所有关键字都小于它, 而右子树的关键字都大于它
5. 所有的叶子节点都位于同一层
6. 每个节点都有索引和数据
```



### B+树的概念

```
	m: 每个树节点最多的有m个查找路径
1. 根节点最少有1个关键字
2. 非根节点最少有m/2个关键字
3. 每个节点最多有m-1个关键字
4. 内部节点只存储索引, 数据都存储在叶子节点
5. 每个叶子节点都有相邻叶子节点的指针, 叶子节点本身按照关键字从小到大顺序连接
6. 父节点存有右子树第一个元素的索引
7. 每个节点的关键字都是按照从小到大的顺序排列, 每个关键字的左子树的所有关键字都小于它, 而右子树的关键字都大于它
8. 叶子节点保存了父节点所有的关键字
```



### B树 & B+树区别

```
	1. B树每个节点都会存储索引和数据; B+树内部节点只存储索引, 数据全部存储在叶子节点中, 形成一个有序链表. 所以B+树更加便于查找; 同时单一节点可以存储的元素更多, 减少了IO次数; 所有的查询都要找到叶子节点, 查询性能是稳定的, 而B树每个节点都可查找到数据, 所以不稳定.
```
